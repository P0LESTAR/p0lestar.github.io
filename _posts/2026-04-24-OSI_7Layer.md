## L1. 물리 계층
- LAN, 케이블 같은 물리적인 매체를 통해 전송 가능한 신호(0,1)로 변환하는 역할.
- 물리 계층의 대표 장치: 증폭기, 허브

## L2. 데이터 링크 계층
> "옆집(인접 노드)과 어떻게 소통할까"

- 물리 계층을 통해 송수신되는 정보의 **오류를 완화**하고 (신뢰성 있는 전송 보장), 기기 간의 물리적인 연결을 관리한다
- 직접 연결된 두 개의 인접한 네트워크 개체 간의 데이터 전송을 담당한다

- 데이터 링크 계층의 전송 단위 (PDU) : **frame**

- 데이터 링크 계층의 대표 장치: 
    - **L2 스위치**: MAC 주소 테이블을 보고 목적지 MAC 주소가 있는 포트로만 데이터를 보내주는 장치
    - **Bridge**(브리지): 두 개의 LAN 세그먼트 연결, MAC 기반 프레임 전달
    - **NIC**(랜카드)

- 데이터 링크 계층의 프로토콜
    - 유선 LAN → Ethernet → MAC 주소 사용
    - 무선 LAN → WiFi → 무선 매체 접근 제어(CSMA/CA)
    - 주소 변환 → **ARP** → IP 주소를 통해 MAC 주소를 찾아내는 프로토콜
    - 루프 방지 → STP → 스위치 간의 무한 루프 차단
    - 네트워크 분할 **VLAN** → IEEE 802.1Q → 물리적인 스위치 하나를 논리적으로 여러 개의 네트워크로 분리

## L3. 네트워크 계층
> "수많은 네트워크에서 목적지까지 어떻게 데이터를 보낼까"

- 여러 개의 노드를 거칠 때마다 경로를 찾아주는 역할
- IP를 사용하여 데이터의 최종 목적지를 식별하고, 수많은 경로 중 최적의 경로를 선택하는 **라우팅**을 수행

- 네트워크 계층의 데이터 단위 (PDU): **Packet**
    - L4(전송 계층)의 세그먼트에 IP 헤더(출발지/목적지 IP 등)가 붙은 형태

- 네트워크 계층의 대표 장치
    - 라우터: 서로 다른 네트워크를 연결하고 패킷의 경로를 결정하는 L3의 핵심 장치.
    - L3 스위치: L2 스위치에 라우팅 기능을 합친 장비(비싸다).

- 네트워크 계층 대표적인 프로토콜
    - **IP**
    - **ICMP** → 오류 보고 및 네트워크 상태 진단하는 프로토콜 
    - IGMP → 멀티캐스트 그룹을 관리하기 위한 프로토콜
    - 라우팅 프로토콜 (RIP, OSPF, BGP)


## L4. 전송 계층
> "어느 프로그램에게, 어떤 방식으로 전달할까?"

- 송신 프로세스와 수신 프로세스 간의 종단 간(End-to-End) 통신을 지원한다
    - 포트 번호를 통한 프로세스 식별
    - 오류 제어
    - 흐름 제어
    - 혼잡 제어

- 전송 계층의 데이터 단위 (PDU)
    - **세그먼트** (Segment): TCP 사용할 때
    - **데이터그램** (Datagram): UDP 사용할 때

- 전송 계층의 대표적인 장치
    - **L4 스위치**(Load Balancer): IP 주소와 포트 번호를 함께 확인하여 트래픽을 분산

- 전송 계층 대표적인 프로토콜
    - **TCP**, **UDP**


## L5. 세션 계층
- 응용 프로그램 간의 통신 세션을 구성하고, 유지하며, 동기화하는 역할
- 주요 기술
    - RPC, NetBIOS


## L6. 표현 계층
- 코드간의 번역을 담당하여 사용자 시스템에서 데이터의 형식상 차이를 다루는 부담을 응용 계층으로부터 덜어준다

- 주요 기능
    - 코덱
    - 변환, 암호화, 압축

## L7. 응용 계층
- 사용자 인터페이스를 제공하고, 응용 프로그램이 네트워크 서비스를 이용할 수 있도록 지원한다

- 응용 계층의 프로토콜
    - HTTP(s)
    - FTP
    - DNS
    - DHCP
    - SMTP/ POP3/ IMAP
    - Telnet/ SSH
